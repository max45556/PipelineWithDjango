General Idea

The Snippet Analyzer is software that allows you to perform various operations on Python snippet. The software is accessible through web APIs developed using Django Rest Framework. The Web Server that provides the services is running on a Docker Container and an application has been designed in Python that allows, through a GUI, to use the server's functions in a simple way. For further information, or for documentation, please contact the following email address: nicolinimassimiliano96@gmail.com

To use the Api you need to register. After registering a user can perform the following functions:
- Manage profile
- Manage personal snippet
- Manage operation on snippet

Below we will better analyze the features and endpoint api

Snippet operation


Below are some uri used to perform operations on the code. For example, you can ask the server to re-indent the code or verify that it is executable. To implement the functions, various libraries have been used which will be explained in detail later. Almost every feature is accessible through three types of requests: a GET request, a POST request, and a PATCH request.

The GET is used when you want to act on a snippet that is stored on the server. For this reason, in the URI it is necessary to specify the ID of the snippet on which to perform the operation. For example, to perform the operation of identifying the programming language of the snippet, which has an id equal to 2, the following request must be made:

GET 127.0.0.1:8000/snippets/2/detect/

The POST is used when you want to act on a snippet which, however, is not stored on the server and which is passed to the server. For this reason, when the request is made through the uri it is necessary to specify in the body a parameter that contains the code to act on, 'code': 'code to analize'. For example, to identify the programming language of the snippet passed to the server, it is necessary to do:

POST 127.0.0.1:8000/snippets/detect/ and in the body you need to insert the snippet

The PATCH works almost identical to Get. The patch is used to perform operations on a snippet saved on the server and the difference from the get is that when the output is obtained, the value obtained is saved in the db. For example, when the snippet programming language identification operation is performed, and the result is obtained, this value is stored in the language field of the instance of this snippet on the db. For example, to perform the operation of identifying the programming language of the snippet, which has an id equal to 2, the following request must be made:

PATCH 127.0.0.1:8000/snippets/2/detect/

if the returned output is Python, the Python value will be entered in the language field of snippet 2.

Detect language snippet


Through this feature it is possible to identify the programming language of a particular snippet. Identification is done through Guesslang, which is open source deep learning software that has been trained with over a million source code files. This supports more than 50 programming languages and detects the correct programming language with greater than 90% accuracy. Guesslang can be used as a command line interface tool or as a Python module.

Request:

- GET 127.0.0.1:8000/snippets/snippet_id/detect
- PATCH 127.0.0.1:8000/snippets/snippet_id/detect/
- POST 127.0.0.1:8000/snippets/detect/

Header:

- Content-Type: application/json
- Authorization: Bearer + Access Token

Body:

- GET REQUEST, empty
- PATCH REQUEST, empty
- POST REQUEST, 'code': 'code_to_analyze'

Standard Response:

200 OK

Data returned:

Language identified

Reindent snippet code

By using it, you agree to cede control over minutiae of hand-formatting. In return, Black gives you speed, determinism, and freedom from pycodestyle nagging about formatting. Blackened code looks the same regardless of the project you're reading. Formatting becomes transparent after a while and you can focus on the content instead. Black makes code review faster by producing the smallest diffs possible.
